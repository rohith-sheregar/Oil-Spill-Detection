# Deep-SAR Oil Spill Detection

This project is an **Oil Spill Detection System** using Deep Learning. Its goal is to analyze **SAR (Synthetic Aperture Radar) satellite imagery** from the ocean and automatically identify (segment) the exact pixels that contain an oil spill.

Because radar images can see through clouds and darkness, they are perfect for monitoring the ocean. However, radar imagery is grayscale and noisy, so this project utilizes a state-of-the-art Neural Network (DeepLabV3+ with a MobileNetV3 backbone and custom scSE Attention blocks) to accurately separate the oil from the seawater.

## Project Structure

Here is a breakdown of the project structure, what each folder holds, and the purpose of each file:

### 1. The `data/` Folder
This folder holds the **SOS (Deep-SAR Oil Spill) dataset**. It contains the raw satellite images and their corresponding "masks" (labels).
* **`images/`**: The raw, grayscale satellite radar pictures of the ocean.
* **`label/`**: The "masks" or ground-truth images. These are black-and-white images where a human expert has manually colored the oil spill white (pixel value 255) and the ocean black (pixel value 0). Our model learns how to generate these masks from the raw images.
* *(Note: This project is configured to train specifically on the Sentinel satellite data subset, split into `train/` and `test/` folders).*

### 2. The `src/` (Source) Folder
This is the core engine of the project containing all the PyTorch code.
* **`dataset.py`**: Handles reading the images and labels from the directories. It performs resizing, normalizes the pixel values, and applies augmentations (like flipping or rotating the images randomly) using Albumentations so the model generalizes well to new data.
* **`model.py`**: Defines the Neural Network architecture. We use `OilSpillDeepLab`, which takes a raw image, passes it through a feature-extractor (`MobileNetV3`), applies an Attention mechanism (`scSEBlock` to focus on important oil patterns and ignore sea waves), and outputs a prediction mask.
* **`utils.py`**: Contains the math and evaluation functions. It includes the custom loss function (`bce_dice_loss`) which calculates the error of the model's predictions, as well as accuracy metrics like `dice_score` and `iou_score` (Intersection over Union) to evaluate performance.
* **`train.py`**: The main training loop. It iterates through the dataset over multiple epochs, feeds the images to the model, calculates the loss, updates the model weights using an optimizer, and evaluates the model on the validation set.

### 3. The Root Files
* **`config.py`**: The master configuration file. Paths, hardware settings, and hyperparameters (like `IMAGE_SIZE`, `BATCH_SIZE`, `LEARNING_RATE`) are defined here. All other modules read from this configuration.
* **`smoke_test.py`**: A diagnostic script to verify that the dataset loads correctly, the model instantiates properly, and the forward passes execute without errors before initiating a full training session.

### 4. The `outputs/` Folder (Generated Automatically)
* **`checkpoints/`**: During training, whenever the model achieves a new best evaluation score, its weights are saved here (e.g., `best_model.pth`). This allows resuming training or performing inference later.
* **`predictions/`**: A directory reserved for saving visual segmentation masks generated by the model on new, unseen satellite imagery.

## Getting Started

To verify your environment and dataset configuration, run the smoke test:
```bash
python smoke_test.py
```

To begin training the model from scratch:
```bash
python -m src.train
```
